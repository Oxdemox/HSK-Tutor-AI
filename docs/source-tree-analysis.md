# Source Tree Analysis

```text
project-root/
├── PRD/                    # Product Requirement Documents and Specs
├── _bmad/                  # BMAD agent tooling and configuration
├── docs/                   # Project documentation generated by BMAD
├── public/                 # Static assets
├── src/                    # Application source code
│   ├── components/         # React UI Components and Views
│   ├── data/               # Static data (curriculum, vocabulary)
│   ├── styles/             # Global CSS and Tailwind definitions
│   ├── utils/              # Helper functions (audio, etc.)
│   ├── App.tsx             # Main application entry point and routing/auth logic
│   ├── firebase.ts         # Firebase initialization
│   ├── index.css           # Global entry CSS
│   └── main.tsx            # React DOM mounting
├── firebase-*.json         # Firebase configuration files
├── package.json            # NPM dependencies and scripts
└── vite.config.ts          # Vite build configuration
```

## Critical Folders
- **`src/components/`**: Contains all the distinct views of the application (e.g., VocabularyView, ProgressView, QuizView, etc.). This is where the UI revamp will primarily take place.
- **`src/data/`**: Holds the curriculum definitions and static HSK data.
- **`src/utils/`**: Contains utility classes, notably `AudioRecorder` and `AudioPlayer` for the AI voice interaction.
